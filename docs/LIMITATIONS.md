# Limitations and Non-claims

PulseWetProbe does not claim:

- certified RWIS performance,
- exact EC measurement,
- exact salinity measurement,
- exact freezing point detection,
- universal soil moisture percentage,
- universal wet/dry thresholds,
- corrosion elimination,
- hardware validation on every Arduino-compatible board.

## Current implementation limits

- Phase 2 includes two-plate and guarded ESP32 touch modes; a true four-plate acquisition API is not yet implemented.
- ESP32 touch availability depends on the exact ESP32 target, Arduino core support and a touch-capable GPIO. Unknown ESP32-family pin maps are not treated as measurement evidence until hardware-tested.
- ESP8266 is treated as a one-ADC board.
- Profile export/import is provided, but persistence backend is left to the sketch/user.
- Hardware validation evidence must be generated by the owner on real boards and probes.

## Correct public wording

Use:

> conductivity trend
> wetness index
> brine-like response trend
> owner-validated calibration
> experimental ice-risk input signal

Do not use:

> certified EC meter
> salinity meter
> road ice detector
> freezing point sensor
> universal moisture percentage

- All advanced diagnostics are heuristic/trend-based indicators until owner-side hardware CSV logs and plots validate them for a specific electrode geometry and environment.
